# weekly_review

## Purpose
Produce a weekly review draft from measurable records.

## Inputs
- All daily logs from the current week
- Expense and income records
- Project updates

## Output
- `reports/weekly/YYYY-WW-weekly-review.md`

## Required Sections
1. Weekly Scorecard
2. What improved
3. What declined
4. Time analysis
5. Money analysis
6. Project progress
7. Repeated bottlenecks
8. Next week priorities
9. System improvements

## Rules
- Use numbers where possible.
- Do not moralize.
- Be concrete.
